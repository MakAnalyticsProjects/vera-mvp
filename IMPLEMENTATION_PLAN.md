I can write the entire schema and migration files locally without an active DB. They just can't be applied until `DATABASE_URL` arrives.

**File structure to add:**

```

```

**Prisma schema (`schema.prisma`):**

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

model Tenant {
  id                  Int       @id @default(autoincrement())
  name                String
  slug                String    @unique
  senderDomain        String?
  brandColor          String?
  briefingTimeLocal   String    @default("07:00")
  briefingTimezone    String    @default("America/Chicago")
  createdAt           DateTime  @default(now())

  users               User[]
  schedules           Schedule[]
  sendLog             SendLog[]
  briefings           Briefing[]
}

model User {
  id          Int       @id @default(autoincrement())
  tenantId    Int
  email       String    @unique
  googleSub   String?   @unique
  name        String?
  imageUrl    String?
  role        String    @default("member")  // 'owner' | 'member'
  createdAt   DateTime  @default(now())

  tenant      Tenant    @relation(fields: [tenantId], references: [id])

  @@index([tenantId])
}

model Schedule {
  id            Int       @id @default(autoincrement())
  tenantId      Int
  cadence       String    // 'daily' | 'weekly' | 'monthly'
  dayOfWeek     Int?      // 0-6 for weekly
  dayOfMonth    String?   // '1'..'28' or 'last' for monthly
  timeLocal     String    // 'HH:mm'
  timezone      String    // IANA, e.g. 'America/Chicago'
  recipient     String
  enabled       Boolean   @default(true)
  nextRunAt     DateTime?
  lastRunAt     DateTime?
  createdAt     DateTime  @default(now())

  tenant        Tenant    @relation(fields: [tenantId], references: [id])
  sendLog       SendLog[]

  @@index([tenantId, enabled, nextRunAt])
}

model SendLog {
  id            Int       @id @default(autoincrement())
  tenantId      Int
  scheduleId    Int?      // nullable — send-now sends have no schedule
  cadence       String
  toEmail       String
  resendId      String?
  pdfBytes      Int?
  status        String    // 'sent' | 'failed' | 'cancelled'
  errorMessage  String?
  sentAt        DateTime  @default(now())

  tenant        Tenant    @relation(fields: [tenantId], references: [id])
  schedule      Schedule? @relation(fields: [scheduleId], references: [id])

  @@index([tenantId, sentAt])
}

model Briefing {
  id                Int       @id @default(autoincrement())
  tenantId          Int
  generatedAt       DateTime  @default(now())
  headline          String
  bodyMd            String    @db.Text
  keyJobs           Json?
  model             String?
  promptTokens      Int?
  completionTokens  Int?

  tenant            Tenant    @relation(fields: [tenantId], references: [id])

  @@index([tenantId, generatedAt(sort: Desc)])
}
```

**Seed (`seed.ts`):**

```ts
import { PrismaClient } from '@prisma/client';
const prisma = new PrismaClient();
async function main() {
  await prisma.tenant.upsert({
    where: { slug: 'priority-roofs-dallas' },
    update: {},
    create: {
      name: 'Priority Roofs · Dallas',
      slug: 'priority-roofs-dallas',
      senderDomain: 'makanalytics.org',
      brandColor: '#B85C2A',
      briefingTimeLocal: '07:00',
      briefingTimezone: 'America/Chicago',
    },
  });
}
main().finally(() => prisma.$disconnect());
```

```ts

export const db = globalThis.__prisma ?? new PrismaClient();

```

## Risk register / things that could go sideways

| Risk | Mitigation |
|---|---|
| GH Actions cron skips slots during platform high-load | 15-min cadence is forgiving; the next run picks up pending schedules with at most ~30 min drift. Acceptable for daily AR briefs. |
| Neon free-tier cold starts (~500ms first query after idle) | Add a `vercel build` step that warms the connection. Acceptable for a single tenant; revisit if we add real users. |
| OpenAI API quota / outage during a briefing run | Briefing fallback to static `composeBriefing` is built into Phase 3 — page never breaks. |
| Auth.js v5 still in beta | Pin to a stable v5 release tag (e.g. `5.0.0-beta.25` or whatever's marked stable at implementation time); fall back to v4 if integration issues block. |
| Prisma client size on Vercel functions (cold start ~100ms hit) | Use Prisma Accelerate or Neon serverless driver if cold-start latency becomes user-visible. Not a v1 concern. |
| GH Actions repo write access lost (you remove me) | Wouldn't break running code, would block future workflow updates. Confirm I keep collaborator access. |
